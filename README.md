# User Management Dashboard (Next.js + Supabase)

## Project Overview

This project is an **Admin User Management Dashboard** built using **Next.js (TypeScript)** and **Supabase (PostgreSQL + Auth + Realtime)**.

The application allows an **admin user** to manage users through a dashboard interface with features like:

* Secure admin authentication
* Importing users from Excel files
* Exporting users to Excel
* Searching and paginating users
* Realtime UI updates
* Role-based access control using Row Level Security (RLS)

This project demonstrates **production-level patterns** for working with Supabase and Next.js.

# Features

## 1. Admin Authentication

Admins can log into the dashboard using **Supabase Auth**.

Features:

* Email/password login
* Session handling
* Dashboard route protection
* Redirect if not authenticated

The dashboard uses **auth state change side effects** to ensure unauthorized users cannot access protected routes.

---

# 2. Protected Dashboard

Only authenticated admins can access the dashboard.

Protection is implemented by:

* Checking Supabase session
* Redirecting unauthorized users
* Enforcing database rules through **RLS policies**

---

# 3. User Table Dashboard

The dashboard displays a table of users with the following fields:

| Field      | Description        |
| ---------- | ------------------ |
| id         | Unique UUID        |
| name       | User name          |
| email      | Email address      |
| phone      | Phone number       |
| role       | User role          |
| created_at | Creation timestamp |

Features:

* Pagination
* Search filtering
* Sorting
* Realtime updates

---

# 4. Search and Pagination (RPC)

User data is fetched using a **PostgreSQL RPC function**.

Example logic:

* Filter users by search text
* Search across name, email, phone, and role
* Paginate results using limit and offset
* Return total row count

This improves performance compared to fetching all rows.

---

# 5. Excel Import

Admins can import users by uploading an **Excel (.xlsx) file**.

### Import Process

1. Admin uploads an Excel file
2. File is parsed using the `xlsx` library
3. Rows are converted to JSON
4. Data is sent to a Supabase **RPC function**
5. Users are inserted into the database

### Expected Excel Format

| name       | email                                 | phone      | role  |
| ---------- | ------------------------------------- | ---------- | ----- |
| John Doe   | [john@mail.com](mailto:john@mail.com) | 9876543210 | user  |
| Jane Smith | [jane@mail.com](mailto:jane@mail.com) | 9876543211 | admin |

Notes:

* UUID is automatically generated by the database
* Duplicate emails can be validated before insertion

---

# 6. Excel Export

Admins can export the current user list as an **Excel file**.

Export process:

1. Fetch user data from Supabase
2. Convert JSON to worksheet
3. Create workbook
4. Trigger browser download

This allows easy sharing or backups of user data.

---

# 7. Realtime Updates

Supabase **Realtime** is used to automatically update the UI when database changes occur.

Events tracked:

* INSERT
* UPDATE
* DELETE

When any user changes:

* Dashboard updates automatically
* No manual refresh required

---

# 8. Role Based Access Control (RLS)

Row Level Security is enabled on the `user_management` table.

Only **admin users** can perform operations such as:

* Insert
* Update
* Delete
* Select

This ensures security even if someone tries to access the database directly.

# How to Run the Project

### 1. Clone the repository

```
git clone <repo-url>
```

---

### 2. Install dependencies

```
npm install
```

---

### 3. Setup environment variables

Create `.env.local`

```
NEXT_PUBLIC_SUPABASE_URL=your_project_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
```

---

### 4. Run development server

```
npm run dev
```

---

### 5. Open application

```
http://localhost:3000
```
